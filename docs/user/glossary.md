# Glossary

## Component

A calculation unit such as equipment, controller, data source, data sink, utility, or user-defined model.

## Node

A component connection point. A chiller is a component; a chilled-water inlet is a node.

## Connection

A directed link from one component output node to another component input node.

## System

A runnable collection of components, connections, public inputs, and public outputs.

## Public Input

An externally supplied input to a system.

## Public Output

An externally visible output from a system.

## Parameter

A named model value used by component logic.

## Parameter Set

A named collection of parameter values for a baseline, scenario, calibration result, or optimization result.

## State

Values carried between evaluations.

## Scenario

A saved input/context case for repeatable runs.

## Runner

The executable that validates and runs project files.

## Worker

The Python process that executes user-defined component code.

## Validation

Checking project structure and comparing model outputs against reference data, depending on context.

## Calibration

Estimating model parameters to match observed data.

## Optimization

Changing decision variables to improve an objective.

## Runtime-Only Package

A delivery package that runs a model without the Studio GUI.
